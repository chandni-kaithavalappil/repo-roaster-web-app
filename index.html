How it works

Static, read-only, GitHub Pages friendly.

1

Fetch signals

Repo metadata, languages, tree, README, package files, server-ish files, CI, tests, deploy hints, and docs clues.

2

Score readiness

Heuristics turn evidence into demo-readiness, documentation, setup, reliability, organization, story, and polish scores.

3

Render the report

A visual dashboard shows stats, risks, gaps, top fixes, and a roast that stays funny without being feral.

Built for prototype/demo readiness reviews. Not a security audit. Does not clone or modify target repos.
